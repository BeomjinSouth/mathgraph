# Agent Docs

Use these files for every new task before coding, during implementation, and when closing out work.

## Files

- `prd.md`: define the problem, goals, scope, and acceptance criteria.
- `implementation_tracking.md`: track status, decisions, progress, blockers, and verification.
- `skills_context.md`: capture relevant skills, repo notes, constraints, and helpful references.
- `implementation_plan.md`: legacy or long-running plan notes; update only when a task needs a fuller roadmap.

## Workflow

1. Start by filling in `prd.md`.
2. Add the execution plan and current state to `implementation_tracking.md`.
3. Record relevant constraints, assumptions, and reusable guidance in `skills_context.md`.
4. Update the tracking file as work changes.
5. When the task is done, leave the three files in a state that makes the next task easier to start.
