# Solid 3D Vertex Labeling And Rotation PRD

## Summary

입체도형(각기둥, 각뿔)을 만들면 정점이 자동으로 이름을 가지며, 점 이름이 도형과 함께 자연스럽게 보여야 한다. 또한 입체도형을 이동하거나 회전할 때 정점과 라벨이 항상 함께 따라와야 한다.

## Problem

- 입체도형의 정점은 내부적으로 점 객체를 사용하지만, 렌더 순서 때문에 도형 선이 점/라벨보다 위에 그려져 이름이 눈에 잘 띄지 않는다.
- 입체도형 자체를 직접 드래그해 이동하는 경험이 부족해서, 사용자가 "도형"과 "정점"이 분리되어 있다고 느끼기 쉽다.
- 각기둥 윗면 점은 프라임(`A'`) 방식으로 이름이 붙어, 사용자가 기대하는 `A, B, C, D...` 순차 라벨과 다르게 보일 수 있다.
- 입체도형 방향을 바꾸기 위해 사용자가 꼭 정점을 하나씩 수동으로 움직여야 해서 회전 조작이 부족하다.

## Goals

- 새로 만드는 입체도형의 정점 이름이 자동으로 붙는다.
- 입체도형 위에서도 정점과 라벨이 안정적으로 보인다.
- 입체도형을 직접 드래그하면 관련 정점들이 함께 이동한다.
- 입체도형을 회전하면 관련 정점과 라벨이 함께 움직인다.
- 정점을 직접 이동해서 입체도형 형태를 바꿔도 라벨이 함께 움직인다.

## Non-Goals

- 전용 회전 핸들이나 별도 회전 패널을 새로 추가하지 않는다.
- 기존 일반 2D 점 라벨 편집 UX를 바꾸지 않는다.
- AI JSON 스키마를 이번 작업에서 확장하지 않는다.

## User Stories

- 사용자는 각기둥/각뿔을 만든 뒤 정점 이름을 바로 보며 설명할 수 있다.
- 사용자는 도형 몸체를 잡고 옮겨도 점과 이름이 함께 따라오는 것을 기대한다.
- 사용자는 입체도형을 선택한 뒤 간단한 조작으로 전체 방향을 회전시키고 싶어 한다.
- 사용자는 정점을 끌어 입체감 방향을 바꿔도 이름이 정점 옆에 붙어 있기를 기대한다.

## Acceptance Criteria

- 각기둥 생성 시 밑면/윗면 점 이름이 자동으로 이어진다.
- 각뿔 생성 시 밑면/꼭대기 점 이름이 자동으로 보인다.
- 캔버스에서 입체도형 선보다 정점/라벨이 위에 그려진다.
- 각기둥/각뿔을 클릭 후 드래그하면 관련 자유점들이 동일 벡터로 이동한다.
- 각기둥/각뿔을 선택한 뒤 `Ctrl+드래그`하면 도형 중심 기준으로 관련 자유점들이 함께 회전한다.
- 위 이동 동작은 undo/redo 히스토리와 함께 동작한다.
- 위 회전 동작도 undo/redo 히스토리와 함께 동작한다.
